# AI Friend Adventure Book Design

Alignment status: 2026-07-06 fixed baseline. The current UI is a playable AI partner island for children, with parent tools separated, drawers preserving state, mobile sticker boards single-column, and child-facing copy centered on wishes, AI drafts, feedback, checking, and child decisions.

## Audience

An 8-year-old child exploring AI at home with a parent. The design should feel like a playful mission book, not a classroom worksheet.

## Visual Direction

- Warm, bright, and imaginative: sky blue, mint, sunny yellow, coral, soft lilac.
- Use rounded illustrated scenes, sticker-like badges, mission cards, and friendly character silhouettes.
- Make the child the hero. AI characters, sprites, pets, and companions are helpers, never the boss.
- Keep text short on visual pages. One main sentence per panel is enough.
- Use visual hooks: game sprites, digital pets, mini-game screens, learning companion dashboard, treasure map paths, badges, stars, batteries, hearts, and progress meters.

## Typography

- Chinese UI/display font stack: `PingFang SC`, `Microsoft YaHei`, Arial, sans-serif.
- Big friendly headings, strong weight, no negative letter spacing.
- Body text must be large enough for a child to read with a parent.

## Components

- Mission page: large illustration, one title, one promise, three playful actions.
- Character card: avatar, name, superpower, one thing the child controls.
- Parent note: quiet bordered panel, smaller and calmer than child-facing content.
- Progress badge: icon-like symbol plus short label.

## Interaction And Print

- The primary artifact is a browser-readable and printable HTML booklet.
- Pages should fit comfortably as vertical cards on screen and print as separate pages when possible.
- Avoid relying on external assets or network fonts.
