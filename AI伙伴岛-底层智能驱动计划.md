# AI伙伴岛底层智能驱动计划

文档对齐状态：2026-07-06 修复版。底层驱动边界不变：AI 草稿先进家长预览；当前页面交互修复已纳入主游戏基线。

## 当前选择

本地家庭玩法继续用 Codex 做底层智能驱动，豆包只预留接口。

```text
AI伙伴岛 HTML 游戏
-> 本地 / N100 Node 服务
-> Codex 导演生成草稿
-> 家长预览
-> 孩子选择、修改、写入成长日记
```

豆包适合以后扩展中文对话、图像生成或给其他家庭做云端轻量版，但当前不是必需依赖。

## 为什么 Codex 更适合本地驱动

- 可以围绕当前项目文件持续迭代。
- 可以生成 HTML、小游戏原型、打印页和说明文档。
- 可以通过本地服务把草稿先交给家长预览。
- 更适合“家长掌控、孩子本地玩”的家庭场景。

## 当前已经具备的能力

- `AI伙伴岛.html` 单页游戏。
- N100 局域网部署。
- 10 个 AI 成长主题。
- 电子宠物向导三步主流程。
- 成长日记、本地进度、星星、徽章。
- 语音小精灵：朗读、听写、简单口令。
- 家长高级设置不会隐藏页面；更多玩法和成长工具箱会保留展开状态。
- 第 4 关前的宠物孵化入口不会跳关；说愿望入口会展开 AI 伙伴对话练习。
- 愿望引擎发明/侦探文案、手机贴纸板单列、英文名即时反馈、伙伴回信跨关刷新已纳入当前页面能力。
- `GET /health` 服务检测。
- `POST /api/codex/level-draft` 生成家长预览草稿。
- `PROVIDER=local` 安全本地草稿模式。
- `PROVIDER=openai` 结构化草稿模式。
- 豆包提示词复制和接口预留。

## 服务接口

### `GET /health`

返回本地导演服务状态。

### `POST /api/codex/level-draft`

输入：

- 主题 id。
- 主题标题。
- 孩子昵称。
- 小精灵昵称。
- 电子宠物昵称。
- 当前成长日记。

输出：

- 故事草稿。
- 孩子小任务。
- 家长引导。
- 奖励建议。

所有输出必须先进家长预览，不能直接覆盖游戏内容。

## 浏览器端规则

- 浏览器不保存 API key。
- 浏览器只发送昵称和成长日记，不发送隐私。
- AI 草稿必须由家长点击采用。
- 孩子采用后仍可以继续修改。

## 豆包预留接口

后续可以加：

```text
POST /api/adapters/doubao/prompt
POST /api/adapters/doubao/result
```

用途：

- 更自然的中文陪伴对话。
- 图片生成。
- 多个孩子一起玩时的云端体验。

边界：

- 豆包输出也必须先进家长预览。
- 不能直接写入成长日记。
- 不能绕过隐私过滤。

## 后续路线

1. 继续优化电子宠物台词和可爱反馈。
2. 增加“AI 能帮我做什么”的示例库。
3. 增加 Codex 生成小游戏原型的家长预览流程。
4. 给作品展增加更清楚的“AI 帮了什么 / 我决定了什么”展示。
5. 在需要时接入豆包图片或对话接口。

## 不做的事

- 不让孩子绕过家长直接使用外部 AI。
- 不做复杂账号、排行、商城。
- 不把 AI 草稿当标准答案。
- 不为了技术复杂度牺牲孩子的上手体验。
