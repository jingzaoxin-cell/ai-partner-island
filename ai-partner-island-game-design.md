# AI 伙伴岛 HTML 闯关游戏设计

文档对齐状态：2026-07-06 修复版。当前版本仍保持单文件 HTML + 本地 Node 导演服务；最新基线已修复家长更多设置、抽屉保持、孵化入口、愿望入口、移动端贴纸板、英文名即时反馈和伙伴回信。

## 目标

做一个可直接在浏览器打开的家庭协作闯关游戏，让 8 岁孩子通过任务、角色、电子宠物、小游戏和作品展认识 AI、使用 AI，并学会判断 AI 的边界。

## 当前版本边界

- 游戏主体仍是单文件 HTML/CSS/JS，打开即可玩。
- 进度保存在浏览器 `localStorage`。
- N100 可作为局域网服务器，统一提供游戏页面和本地 Codex 导演 API。
- Codex 是主要智能驱动，服务端支持 `PROVIDER=local` 和 `PROVIDER=openai`。
- AI 生成内容先进入家长预览，家长采用后才写入作品记录。
- 豆包只保留为可选接口，用于以后给其他小朋友、其他家庭或图像/聊天场景接入。
- 语音小精灵是轻量辅助：朗读故事、听孩子说作品记录、识别少量口令。

## 游戏结构

- 玩家身份：孩子是"小队长"。
- AI 伙伴：灵灵是小精灵，豆豆是电子宠物，小问号是学习伴侣。
- 关卡数量：10 关。
- 每关包含：故事开场、孩子任务、家长任务、Codex 导演指令、可选豆包接口提示词、孩子作品记录、通关按钮。
- 奖励系统：星星、徽章、宠物心情值、好奇值。
- 最终目标：家庭作品展，孩子讲清楚"我做了什么、AI 帮了什么、我自己决定了什么"。
- 每关导引：任务导航精灵把流程拆成 5 步。
- 语音能力：读故事、下一步、上一步、去写记录、请 Codex 导演、完成本关、打开作品展。

## Codex 集成策略

主线是"本地玩 + Codex 驱动"。当前版本同时支持两层：

1. 人工中转：HTML 根据当前关卡和孩子输入生成 Codex 指令，家长复制到 Codex 使用。
2. 本地导演：HTML 请求 N100/本地 Node 服务，由服务端生成结构化草稿。

本地导演规则：

- HTML 只向本地服务发送非隐私游戏状态。
- 浏览器不保存 API key。
- 所有 AI 草稿先进入家长预览。
- 家长点击"采用到作品记录"后，孩子继续修改成自己的说法。
- 豆包适配层保持可插拔，不进入主流程。

## 验收重点

- 孩子能从第 1 关玩到第 10 关。
- 每关都有可复制 AI 提示词。
- 通关后能解锁下一关。
- 刷新页面后进度还在。
- 可以重置进度。
- 手机和桌面都能阅读和操作。
- 语音小精灵在支持的浏览器里能朗读和听写；不支持时不影响点击玩法。
- 本地和 N100 服务端测试通过。
